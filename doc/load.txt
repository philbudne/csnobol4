1. Intruduction

External functions are loaded via the LOAD() function.  LOAD() is
passed a prototype which names the function and describes the number
of, and types of the arguments the function will be passed, as well as
the return type.  The named function will first be searched for in a
table built in at compile time (see "Poor Man's LOAD"), if it is not
found, the second argument to LOAD is used as the object file (or
library) to load the function from. If no second argument is given,
the value of the SNOLIB environment variable will be used.  If the
SNOLIB environment variable is not set, a compiled in default path
will be used (by default /usr/local/lib/snobol4/snolib.a) -- see
SNOLIB_DIR AND SNOLIB_A in include/path.h.

examples:
	LOAD("SQRT(REAL)REAL")
	LOAD("DELETE(STRING)STRING")
	LOAD("FILE(STRING)STRING")
	LOAD("FORK()INTEGER")
	LOAD("GETENV(STRING)STRING")
	LOAD("RENAME(STRING,STRING)STRING")
	LOAD("SPRINTF(STRING,INTEGER)STRING")
	LOAD("SYSTEM(STRING)INTEGER")

2. Creating a C function to be loaded.

An external C function should be declared as follws;

    #include "h.h"
    #include "snotypes.h"
    #include "macros.h"
    #include "load.h"

    int
    NAME( LA_ALIST ) LA_DCL
    {
    }


The function should use one of the following macros to return a value
(the type of which should be compatible with the type in the prototype
passed to the LOAD() function;

	RETINT(i);			/* return integer */
	RETREAL(a);			/* return real */
	RETNULL;			/* return null string, success */
	RETSTR(buf,len);		/* return non-null string */
					/* depends on snolib/retstr.c */
	RETFAIL;			/* return failure */

Arguments can be accessed via the following macros;

	LA_INT(N)	access the n'th argument as an integer (int_t)
	LA_REAL(N)	access the n'th argument as a real (real_t)
	LA_PTR(N)	access the n'th argument as a pointer (void *)
			pass to the getstr() function from snolib/getstr.c
			to retrieve a string, ie;
				getstr(LA_PTR(n), buffer, sizeof(buffer));

	LA_TYPE(N)	returns the datatype of the n'th argument
			NOTE: this should always be the type as declared
			in the function prototype given to LOAD()!!!

		I	arg is an integer of type int_t
		R	arg is a real of type real_t
		S	arg references a string or "natural variable"
			constructed of 4 decsriptors, plus the string.
			(see the getstring() function)

			The D_V() field of the first descriptor
			contains the string length.  The string begins
			after the 4th descriptor (at D_A(dp) + BCDFLD)

		user datatypes begin at 100 and appear as an array of
		descriptors located by D_A(dp).  The first descripor
		is a "title" descriptor whose D_V() field indicates
		how many descriptors follow.


Dynamicly loaded functions have no access to any global data or
functions in the SNOBOL4 system.  Any libraries required to load the
function must be mentioned explcitly in the second argument to LOAD().

3. ``Static'' (or ``Poor Man's'') Loading

On systems (at this time anything other than BSD flavored Un*x) which
don't implement the load() library function (and even on systems which
do), external functions may be linked into the snobol4 system image at
build time.

Step 1.

	Add a line to the top level file "pml.h" which describes the
	function name and it's entry point.  If the names are the same, use
	PMLFUNC(NAME), else use PMLFUNC2("NAME", entry).  pml.h
	econtains xamples for many of the functions in snolib/*.c

Step 2.

	Add the paths of the object file(s) in which the functions
	reside to the PML_OBJ variable in your config.h file. ie;

	PML_OBJ=system.o

	If any special libraries are required, you can specify them
	at the end of PML_OBJ, or add them to CONFIG_LDFLAGS.

Step 3.

	Make sure the named object files exist.

Step 4.

	Run "make"
